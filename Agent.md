## 학습 순서

### 1단계 : 개념 설명
- 해당 일차의 핵심 함수/개념을 설명한다
- 각 개념마다 **기본 예제 코드**와 **시험에서 자주 쓰는 패턴**을 함께 제시한다
- Colab에서 바로 실행할 수 있는 형태로 제공한다
- 질문이 나오면 즉시 답변하고 넘어간다

### 2단계 : 연습문제 출제

일차별 문제 수는 아래 기준을 따른다.

| 일차 | 문제 수 | 이유 |
|------|------|------|
| Day 1~3 (Python/Pandas 기초) | 7문제 | 개념당 1~2문제, 단순 패턴 반복 |
| Day 4~7 (Pandas 핵심 조작) | 8~10문제 | 단일 조작 → 복합 조작 순서로, 개념 커버리지 확보 |
| Day 8~9 (고급 조작 + 기출) | 8~10문제 | 피벗/상관/기출 복합 패턴 포함 |
| Day 10~13 (모델 + 통계) | 6~8문제 | 모델 셋업 코드가 길어서 문제당 분량이 많음 |
| Day 14 (기출 실전) | 실전 1세트 | 1+2+3유형 시간 재고 풀기 |

- 난이도는 **기본 → 응용 → 복합** 순서로 구성한다
- **`datamanim_type1_reference.md`의 개념별 문제 매핑을 참고**해 실제 시험 출제 패턴 반영
  - 해당 일차 개념에 매핑된 문제 번호를 확인하고, 동일한 조작 패턴을 연습문제에 녹인다
- 복합 문제(응용 이상)는 반드시 **조건 필터 → 집계/조작 → 정렬 → N번째 값 출력** 흐름을 포함한다

### 3단계 : 풀이 평가
- 제출된 풀이를 문제별로 ✅ / ❌ / △ 로 평가한다
- 오답은 **이유를 설명**하고 **모범답안**을 제시한다
- △ (부분 점수)는 맞은 부분과 개선할 부분을 구분해서 설명한다
- 추가로 알아두면 좋은 내용(더 나은 방식 등)도 함께 안내한다

### 4단계 : 문서 업데이트
연습문제 완료 후 아래 파일들을 업데이트한다.

| 파일 | 업데이트 내용 |
|------|-------------|
| `practice/dayN.md` | 해당 일차 연습문제 + 최종 풀이 저장 |
| `복습노트.md` | 해당 일차에서 배운 핵심 내용 추가 (Day별 누적) |
| `치트시트.md` | 자주 틀리는 패턴 / 함정 발견 시 추가 (누적) |
| `공부계획.md` | 해당 일차 완료 ✅ 표시 |
| `오답노트.md` | 틀렸거나 헷갈렸던 내용 추가 (아래 기준 참고) |
| `_sidebar.md` | 새 day 파일 추가 시 연습문제 + 복습풀이 섹션에 링크 추가 |

> **중요**: `_sidebar.md`를 업데이트하지 않으면 Docsify 배포 페이지에 해당 일차가 표시되지 않는다.

**오답노트 기록 기준**
- ❌ 오답: 틀린 이유 + 올바른 코드
- △ 부분 정답: 빠뜨린 부분 + 보완 코드
- 개념 질문 중 "몰랐다"고 한 내용 (예: round(), copy(), axis 등)

복습 풀이(이전 일차 재풀이)는 `practice/`가 아닌 **`review/`** 에 저장한다.

| 파일 | 저장 내용 |
|------|---------|
| `review/dayN_review.md` | N일차 개념을 다시 풀어본 복습 풀이 |

## _sidebar.md 업데이트 규칙

새로운 일차(dayN) 파일을 추가하면 `_sidebar.md`에 아래 두 항목을 추가한다:

```markdown
# 1유형 — 연습문제 섹션에 추가:
  - [Day N — 주제명](practice/dayN.md)

# 복습 풀이 섹션에 추가:
  - [Day N](review/dayN_review.md)
```

커밋 후 GitHub Pages에 반영되면 배포 페이지에서 확인 가능.

## 복습노트 작성 원칙
- 함수 설명보다 **코드 중심**으로 정리
- 시험에서 실제로 쓰는 패턴 위주
- 주의사항/함정은 `>` 인용 블록으로 강조
