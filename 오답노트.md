## Day 9 — NumPy + 백분위수 + 복합 필터 (2026-06-15)

### transform 없이 그룹 집계값으로 행별 비교 시도 (Q7)
- **실수**: `group_median = df.groupby('dept')['score'].median()` → 3행짜리 Series라 df(15행)와 비교 불가
- **이유**: `groupby().median()`은 그룹당 1행 반환 — 행별 비교에 쓸 수 없음
- **정답**: `transform`으로 각 행에 자기 그룹 값을 붙인 뒤 비교
```python
df['dept_median'] = df.groupby('dept')['score'].transform('median')
above = df[df['score'] >= df['dept_median']]
print(above.groupby('dept')['salary'].mean().round(2))  # ✅
```
> `groupby().median()` → N행 (그룹 수)  
> `groupby().transform('median')` → 원본과 같은 행 수 (행별 자기 그룹 값)

### 조건부 컬럼 생성 패턴 — np.where vs apply (Q8)
- **np.where**: 빠르고 간결 — 시험 우선순위 1순위
- **apply**: 내부적으로 for문과 동일, 복잡한 조건일 때 가독성 좋음
```python
# np.where — 추천
df['travel_type'] = np.where(df['distance'] >= q70, 'long', 'short')

# apply + lambda — 대안
df['travel_type'] = df['distance'].apply(lambda x: 'long' if x >= q70 else 'short')
```
> `apply`는 각 행의 값을 x로 하나씩 꺼내 함수 실행 → for문과 동일한 동작  
> 조건이 단순하면 `np.where`, 조건이 복잡하면 `apply` 선택

### abs() 누락 — 차이의 절댓값 (Q3)

```python
print(round(abs(mean1 - mean2), 2))  # ✅
```

### df.corr() 단독 사용 시 에러 + numeric_only 헷갈림
- **실수**: `df.corr()` — 문자열 컬럼 섞여있으면 에러
- **정답**: `numeric_only=True` 옵션 필수

- 음수: 한쪽이 오르면 다른쪽이 내려가는 관계

## Day 7 — 문자열 처리 (2026-06-14)

### str.extract 패턴 정리
- **어려운 이유**: 정규식 기호를 외워야 하고, 괄호()의 역할이 낯섦
- **핵심 규칙**: 괄호 `()` 안이 추출할 부분

```python
# 자주 쓰는 패턴 4개만 외우기
df['col'].str.extract(r'(\d+)')      # 숫자 추출        'A123' → '123'
df['col'].str.extract(r'(\D+)')      # 숫자 아닌 것     'A123' → 'A'
df['col'].str.extract(r'([A-Z]+)')   # 대문자 알파벳    'A1'   → 'A'
df['col'].str.extract(r'([a-z]+)')   # 소문자 알파벳    'Ab1'  → 'b'
```

- `r''` — raw string. 백슬래시를 문자 그대로 전달 (정규식에 필수)
- `{4}` — 정확히 N개: `\d{4}` = 숫자 정확히 4개
- 결과는 DataFrame → 컬럼으로 바로 할당 가능
- **대안**: 분리/추출이 간단하면 `str.split().str[N]`이 더 쉬움

## Day 5 — 그룹/집계 + 출력 패턴

### reset_index 괄호 누락 + drop=True 빠뜨리기
- **실수**: `df['sales'].sort_values(ascending=False).reset_index.iloc[2]`
- **이유 1**: `reset_index`는 메서드 — 괄호 `()` 없으면 함수 객체 자체를 반환
- **이유 2**: `drop=True` 없으면 원본 인덱스가 컬럼으로 남아 `iloc[2]`가 (index, sales) 두 값을 출력
- **정답**: Series는 `reset_index` 없이 `iloc`만 써도 됨
```python
df['sales'].sort_values(ascending=False).iloc[2]  # ✅ 가장 간단
df['sales'].sort_values(ascending=False).reset_index(drop=True).iloc[2]  # ✅ drop=True 필수
```

### groupby 결과를 df에 직접 할당하려 함 (Q6)
- **실수**: `df.copy()` 후 groupby 결과를 `df['sales']` 컬럼에 할당
- **이유**: groupby 결과는 5행(region 5개), df는 20행 — 크기 불일치
- **정답**: groupby 결과 자체에서 `reset_index()` 후 `iloc`으로 꺼내기
```python
result = df.groupby('region')['sales'].mean().sort_values(ascending=False).reset_index()
print(result.iloc[1]['region'])  # ✅
```

### transform 대신 mean() 직접 사용 (Q10)
- **실수**: `df['rating'] >= df.groupby('gender')['rating'].mean()`
- **이유**: `.mean()`은 그룹당 1행(인덱스: M/F) — df['rating'](인덱스: 0~19)과 크기/인덱스 불일치로 비교 불가
- **정답**: `transform('mean')` — 원본과 같은 길이로 각 행에 그룹 평균을 붙여줌
```python
df.groupby('gender')['rating'].transform('mean')  # ✅ 행별로 자기 그룹 평균
```
