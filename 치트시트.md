## 3. transform vs groupby().agg() — 가장 자주 틀리는 패턴

| | 반환 | 언제 |
|---|---|---|
| `groupby().mean()` | 그룹 수만큼 (N행) | 그룹별 결과만 필요할 때 |
| `groupby().transform('mean')` | 원본과 같은 행 수 | 행별 비교할 때 |

```python
# ❌ 크기 불일치 — 비교 불가
df[df['score'] >= df.groupby('dept')['score'].mean()]

# ✅ transform으로 각 행에 그룹값 붙이기
df['dept_mean'] = df.groupby('dept')['score'].transform('mean')
df[df['score'] >= df['dept_mean']]
```

## 6. 2유형 전처리 패턴 (Day10~)

### 전체 흐름

```
1. 데이터 로드          pd.read_csv()
2. 결측치 처리          fillna / SimpleImputer
3. 인코딩               get_dummies / LabelEncoder  ← split 전
4. X, y 분리            X = df.drop('target') / y = df['target']
5. train_test_split     test_size, random_state, stratify(분류만)
6. 스케일링             fit_transform(train) / transform(test)  ← split 후
7. 모델 학습            model.fit(X_train, y_train)
8. 예측                 model.predict(X_val)
9. 평가                 accuracy/f1/roc_auc (분류) | RMSE/MAE (회귀)
10. CSV 제출            result.to_csv('result.csv', index=False)
```

### 스케일링 — split 필요 여부 구분

| | split 필요 | 적용 대상 | 이유 |
|---|---|---|---|
| 문제에서 train 기준 fit 요구 | O | train/test 분리 후 | 데이터 누수 방지 |
| 문제에서 그냥 변환만 요구 | X | 전체 df에 바로 적용 | split 언급 없으면 전체 |

```python
# split 필요한 경우
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled  = scaler.transform(X_test)   # fit_transform 금지!

# split 불필요한 경우
scaler = MinMaxScaler()
scaled = scaler.fit_transform(df[['col1', 'col2']])
scaled_df = pd.DataFrame(scaled, columns=['col1', 'col2'])
```

> **핵심**: 문제에 split 언급 있으면 split 후, 없으면 전체 df에 바로 적용

df['col_enc'] = le.fit_transform(df['col'])

# 분할 — 분류는 stratify 필수
